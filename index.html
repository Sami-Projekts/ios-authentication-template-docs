🎯 Getting Started

🎉 Two Ways to Get Started

Choose the approach that works best for you:

📱 Use Complete Template
🔧 Integrate into Existing Project

Option 1: Complete Template (Recommended)

Perfect for new projects or when you want a complete authentication system.

  1. Download the template and open in Xcode 15+
  2. Configure Firebase (see detailed setup below)
  3. Customize branding in AppConfig.swift and AppTheme.swift
  4. Run and test all authentication flows
  5. Build your app features on top of the authentication system

Option 2: Integrate into Existing Project

Add professional authentication to your existing iOS app.

  1. Copy core files (ViewModels, Models, Services, Utilities)
  2. Install dependencies (Firebase, GoogleSignIn)
  3. Integrate views into your app's navigation flow
  4. Configure Firebase and authentication providers
  5. Test integration and customize as needed

🛠️ Complete Setup Guide

Prerequisites

  • Xcode 15.0 or later
  • iOS 15.0+ deployment target
  • Swift 5.9 or later
  • Active Apple Developer Account
  • Google Developer Account (for Google Sign-In)

Step 1: Download and Open Project

📱 What you'll do:

Download the iOS Authentication Template from CodeCanyon and open it in Xcode. This template includes all the necessary files and structure for a complete authentication system.

🔧 Detailed Instructions:

  • Download the template ZIP file from your CodeCanyon purchase
  • Extract the ZIP file to your desired location
  • Open Xcode 15 or later
  • Navigate to the extracted folder
  • Double-click the .xcodeproj file to open the project

✅ What you'll see:

The project will open with a complete folder structure including Views, ViewModels, Models, Services, and Utilities folders, all organized following MVVM architecture.

Step 2: Add Swift Package Dependencies

📦 What you'll do:

Add the required Firebase iOS SDK and Google Sign-In packages to your project using Swift Package Manager.

🔧 Detailed Instructions:

  • In Xcode, select your project in the navigator
  • Click on your target under "TARGETS"
  • Go to the "Package Dependencies" tab
  • Click the "+" button to add a new package
  • You'll see the "Add Package Dependency" window

📋 Required packages:

You'll need to add these two packages in the next steps:

  • Firebase iOS SDK:
    https://github.com/firebase/firebase-ios-sdk
  • Google Sign-In iOS:
    https://github.com/google/GoogleSignIn-iOS

Xcode Project Overview - File structure and organization

Step 3: Add Firebase iOS SDK

🔥 What you'll do:

Add the Firebase iOS SDK URL to your project to enable Firebase authentication and Firestore database functionality.

🔧 Detailed Instructions:

  • In the "Add Package Dependency" window, paste the Firebase URL
  • Click "Add Package" and wait for Xcode to resolve the package
  • In the package selection window, choose these Firebase products:

📦 Required Firebase Products:

  • ✅ FirebaseAuth - For user authentication
  • ✅ FirebaseFirestore - For user data storage
  • ✅ FirebaseStorage - For profile pictures (optional)
📋 Copy Firebase URL: https://github.com/firebase/firebase-ios-sdk

Adding Package Dependencies in Xcode Project Settings

Step 4: Select Firebase Products

🎯 What you'll do:

Choose the specific Firebase products your authentication system needs.

🔧 Detailed Instructions:

  • After Xcode resolves the Firebase package, you'll see a list of available products
  • Select FirebaseAuth - Essential for authentication
  • Select FirebaseFirestore - For storing user data and usernames
  • Optionally select FirebaseStorage - For profile pictures
  • Click "Add Package" to add these to your project

⚠️ Important:

Don't add all Firebase products as they will increase your app size. Only add what you need!

Entering Firebase iOS SDK repository URL

Step 5: Add Google Sign-In SDK

🌐 What you'll do:

Add the Google Sign-In iOS SDK to enable Google authentication in your app.

🔧 Detailed Instructions:

  • Click the "+" button again to add another package
  • Paste the Google Sign-In iOS SDK URL
  • Click "Add Package" and wait for resolution
  • Select the GoogleSignIn product
  • Click "Add Package" to complete the installation
📋 Copy Google Sign-In URL: https://github.com/google/GoogleSignIn-iOS

✅ Verification:

After installation, you should see both Firebase and GoogleSignIn in your Package Dependencies list.

Selecting Firebase products for authentication and data storage

Step 6: Create Firebase Project

🎯 What you'll do:

Go to Firebase Console and create a new project for your iOS app.

🔧 Detailed Instructions:

  • Open https://console.firebase.google.com/ in your browser
  • Click 'Create a project' button
  • Enter your project name (e.g., 'MyApp Authentication')
  • Choose whether to enable Google Analytics (recommended)
  • Accept Firebase terms and click 'Create project'
📋 Copy Firebase Console URL: https://console.firebase.google.com/

📝 Important Note:

Choose a descriptive project name as you cannot change it later.

Creating a new Firebase project for your iOS app

Step 7: Add iOS App to Firebase

🎯 What you'll do:

Register your iOS app with Firebase to establish the connection.

🔧 Detailed Instructions:

  • In your Firebase project, click the iOS icon to add iOS app
  • Enter your Bundle ID exactly as shown in Xcode
  • Add an app nickname (optional)
  • Leave App Store ID empty for now
  • Click 'Register app'

📝 Important Note:

Bundle ID must match exactly between Xcode and Firebase.

Adding iOS app to Firebase project and configuring Bundle ID

Step 8: Download GoogleService-Info.plist

🎯 What you'll do:

Download the Firebase configuration file that connects your app to Firebase.

🔧 Detailed Instructions:

  • Click 'Download GoogleService-Info.plist' button
  • Save the file to your Downloads folder
  • This file contains your Firebase project configuration
  • Keep this file secure - it contains API keys
  • Click 'Next' to continue

📝 Important Note:

Never commit this file to public repositories as it contains sensitive data.

Downloading Firebase configuration file

Step 9: Authentication Methods Overview

🎯 What you'll do:

See which authentication methods we'll enable in Firebase Console.

🔧 Detailed Instructions:

  • Navigate to Firebase Console → Authentication → Sign-in method
  • You'll see a list of available authentication providers
  • We will enable three main methods: Email/Password, Google, and Apple
  • These three methods provide comprehensive authentication options
  • Each method will be configured in the following steps

📝 Important Note:

Email/Password provides basic authentication, Google offers easy social login, and Apple Sign-In ensures privacy compliance.

Available authentication methods in Firebase Console

Step 10: Enable Email/Password Authentication

🎯 What you'll do:

Enable email and password sign-in for your users.

🔧 Detailed Instructions:

  • In Sign-in method tab, find 'Email/Password' provider
  • Click on 'Email/Password' to configure it
  • Toggle 'Enable' switch to turn it on
  • Optionally enable 'Email link' for passwordless sign-in
  • Click 'Save' to apply changes

📝 Important Note:

Email/Password is the most basic authentication method and works on all platforms.

Enabling Email/Password authentication in Firebase

Step 11: Enable Google Sign-In

🎯 What you'll do:

Configure Google Sign-In provider for easy user authentication.

🔧 Detailed Instructions:

  • Find 'Google' in the Sign-in providers list
  • Click on 'Google' to configure it
  • Toggle 'Enable' switch
  • Set your project support email address
  • Click 'Save' to enable Google Sign-In

📝 Important Note:

Google Sign-In requires additional setup in Xcode with URL schemes.

Enabling and configuring Google Sign-In provider

Step 12: Enable Apple Sign-In

🎯 What you'll do:

Configure Apple as a sign-in provider in Firebase Console.

🔧 Detailed Instructions:

  • Find 'Apple' in the providers list and click on it
  • Toggle 'Enable' switch to activate Apple Sign-In
  • Configure your App ID and Team ID from Apple Developer Portal
  • Click 'Save' to enable Apple Sign-In authentication

📝 Important Note:

Apple Sign-In provides the highest level of user privacy and security.

Enabling Apple Sign-In provider in Firebase

Step 13: Verify Authentication Methods

🎯 What you'll do:

Confirm all three authentication methods are properly enabled.

🔧 Detailed Instructions:

  • Check that Email/Password shows 'Enabled' status
  • Verify Google Sign-In shows 'Enabled' status
  • Confirm Apple Sign-In shows 'Enabled' status
  • All three methods should have green checkmarks
  • Your authentication setup is now ready

📝 Important Note:

Having multiple authentication options gives users flexibility in how they sign up.

All three authentication methods successfully enabled

🎯 What you'll do:

Download the GoogleService-Info.plist file for your iOS app.

🔧 Detailed Instructions:

  • In the 'Your apps' section, find your iOS app

  • Save the file to your Downloads folder

📝 Important Note:

Downloading GoogleService-Info.plist configuration file

Step 20: Add GoogleService-Info.plist to Xcode

🎯 What you'll do:

Add the Firebase configuration file to your Xcode project and copy the REVERSED_CLIENT_ID.

🔧 Detailed Instructions:

  • Drag GoogleService-Info.plist from Downloads into Xcode project navigator
  • Make sure 'Add to target' is checked for your app target
  • Open the GoogleService-Info.plist file in Xcode
  • Find and copy the 'REVERSED_CLIENT_ID' value
  • Keep this value - you'll need it for URL scheme configuration

📝 Important Note:

The REVERSED_CLIENT_ID is essential for Google Sign-In to work properly.

Adding GoogleService-Info.plist to Xcode and copying REVERSED_CLIENT_ID

Step 21: Configure URL Scheme for Google Sign-In

🎯 What you'll do:

Set up URL scheme in Xcode to handle Google Sign-In callbacks.

🔧 Detailed Instructions:

  • In Xcode, select your project, then your target
  • Go to 'Info' tab and scroll down to 'URL Types'
  • Click '+' to add a new URL Type
  • In 'URL Schemes', paste your REVERSED_CLIENT_ID
  • In 'Identifier', enter 'google signin' for description

📝 Important Note:

This URL scheme allows Google Sign-In to return to your app after authentication.

Configuring URL scheme with REVERSED_CLIENT_ID for Google Sign-In

Step 22: Copy CLIENT_ID from GoogleService-Info.plist

🎯 What you'll do:

Find and copy the CLIENT_ID value for Firebase configuration.

🔧 Detailed Instructions:

  • Go back to GoogleService-Info.plist in Xcode
  • Find the 'CLIENT_ID' key (different from REVERSED_CLIENT_ID)
  • Copy the CLIENT_ID value
  • This will be used for Firebase Google Sign-In configuration
  • Keep this value for the next step

📝 Important Note:

CLIENT_ID and REVERSED_CLIENT_ID are different values used for different purposes.

Copying CLIENT_ID from GoogleService-Info.plist

🎯 What you'll do:

Configure CLIENT_ID in your app target for Firebase Google Sign-In.

🔧 Detailed Instructions:

  • This connects your app to Google Sign-In services
  • Verify the CLIENT_ID is correctly entered

📝 Important Note:

This CLIENT_ID is required for Firebase to authenticate with Google services.

Adding CLIENT_ID to app target configuration

🔧 Troubleshooting Guide

Common Issues & Solutions

🚫 Google Sign-In Not Working

Symptoms: Google Sign-In button doesn't respond or shows error

Solutions:

  • ✅ Verify REVERSED_CLIENT_ID in URL schemes matches GoogleService-Info.plist
  • ✅ Ensure GoogleService-Info.plist is added to project target
  • ✅ Check Google Sign-In is enabled in Firebase Console
  • ✅ Test on physical device (not simulator)
  • ✅ Verify Bundle ID matches exactly between Xcode and Firebase

🍎 Apple Sign-In Failing

Symptoms: Apple Sign-In shows error or doesn't appear

Solutions:

  • ✅ Check "Sign in with Apple" capability is added in Xcode
  • ✅ Verify App ID is configured in Apple Developer Portal
  • ✅ Test on physical device with valid Apple ID
  • ✅ Ensure proper provisioning profile is selected
  • ✅ Check Apple Sign-In is enabled in Firebase Console

📧 Email Verification Issues

Symptoms: Verification emails not received or links don't work

Solutions:

  • ✅ Check spam/junk folder in email client
  • ✅ Verify Firebase email templates are properly configured
  • ✅ Check authorized domains in Firebase Console
  • ✅ Ensure Firebase project is properly set up
  • ✅ Test with different email providers (Gmail, iCloud, etc.)

🔥 Firebase Connection Issues

Symptoms: App crashes on startup or Firebase functions don't work

Solutions:

  • ✅ Verify GoogleService-Info.plist is properly added to project
  • ✅ Check Bundle ID matches exactly in all configurations
  • ✅ Ensure Firebase.configure() is called in AppDelegate
  • ✅ Verify Firebase products are properly imported
  • ✅ Check Xcode console for specific Firebase error messages
