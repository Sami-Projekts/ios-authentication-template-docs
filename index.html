Which Xcode version is needed?

Recommended: Xcode 15.0 or later, you can download from:

  • Mac App Store
  • Apple Developer Portal

Requirements:

  • macOS 13.0 or later
  • iOS 15.0+ deployment target
  • Swift 5.9 or later

How to Open The Project in Xcode?

  • Extract the downloaded ZIP file
  • Navigate to the project folder
  • Double-click on Authentication-Template.xcodeproj
  • Xcode will open with the complete project structure
  • Wait for Xcode to index the project files

Project Structure:

  • App/ - Main app configuration
  • Views/ - SwiftUI views for authentication
  • ViewModels/ - MVVM architecture logic
  • Models/ - Data models
  • Services/ - Firebase and authentication services
  • Utilities/ - Helper functions and configurations

How to change the Bundle ID?

You need to change the Bundle ID to your own unique identifier:

  • In Xcode, select your project in the navigator
  • Click on your target under "TARGETS"
  • Go to the "General" tab
  • Find "Bundle Identifier" in the Identity section
  • Change it to your unique ID (e.g., com.yourcompany.authapp)
  • Make sure it's unique and follows reverse domain notation

Important: This Bundle ID must match exactly in Firebase and Apple Developer Portal!


How to change app name?

  • Open AppConfig.swift file
  • Find the appName property
  • Change static let appName = "Your App Name"
  • For display name, go to Xcode target settings ‚Üí General ‚Üí Display Name

How to setup Firebase?

Step 1: Create Firebase Project

  • Go to Firebase Console
  • Click "Create a project"
  • Enter your project name
  • Disable Google Analytics (optional)
  • Click "Create project"

Step 2: Add iOS App

  • Click "Add app" and select iOS
  • Enter your Bundle ID (same as in Xcode)
  • Enter App nickname (optional)
  • Click "Register app"

Step 3: Download Configuration File

  • Download GoogleService-Info.plist
  • Drag it into your Xcode project root
  • Make sure "Add to target" is checked
  • Click "Add"

Step 4: Enable Authentication

  • In Firebase Console, go to Authentication
  • Click "Get started"
  • Go to "Sign-in method" tab
  • Enable "Email/Password"
  • Enable "Google" (for Google Sign-In)
  • Enable "Apple" (for Apple Sign-In)

Step 5: Setup Firestore Database

  • Go to Firestore Database
  • Click "Create database"
  • Choose "Start in production mode"
  • Select your preferred location
  • Click "Done"

How To Setup Google Sign-In?

Step 1: Configure in Firebase

  • In Firebase Console ‚Üí Authentication ‚Üí Sign-in method
  • Click on "Google"
  • Enable Google Sign-In
  • Set your support email
  • Click "Save"

Step 2: Add URL Scheme

  • In Xcode, go to your target ‚Üí Info tab
  • Expand "URL Types"
  • Click "+" to add new URL Type
  • In "URL Schemes", paste your REVERSED_CLIENT_ID from GoogleService-Info.plist
  • Set Identifier to "google-signin"

Step 3: Test Google Sign-In

  • Build and run on a physical device
  • Google Sign-In doesn't work on simulator
  • Tap "Sign in with Google" button
  • Complete the authentication flow

How To Setup Apple Sign-In?

Step 1: Apple Developer Portal

  • Go to Apple Developer Portal
  • Navigate to Certificates, Identifiers & Profiles
  • Create or edit your App ID
  • Enable "Sign In with Apple" capability
  • Save the configuration

Step 2: Xcode Configuration

  • In Xcode, go to Signing & Capabilities
  • Click "+ Capability"
  • Add "Sign in with Apple"
  • The capability will appear in your project

Step 3: Firebase Configuration

  • In Firebase Console ‚Üí Authentication ‚Üí Sign-in method
  • Click on "Apple"
  • Enable Apple Sign-In
  • Configure Service ID and Team ID
  • Click "Save"

How to configure app settings?

Open AppConfig.swift to customize your app:

Basic Settings

  • appName - Your app display name
  • appVersion - App version number
  • minimumPasswordLength - Password requirements
  • requireEmailVerification - Email verification toggle

Authentication Options

  • enableAppleSignIn - Enable/disable Apple Sign-In
  • enableGoogleSignIn - Enable/disable Google Sign-In
  • enableEmailSignIn - Enable/disable email authentication

Username Settings

  • usernameMinLength - Minimum username length
  • usernameMaxLength - Maximum username length

How to customize theme?

Open AppTheme.swift to change colors and styling:

Colors

  • primaryColor - Main app color
  • secondaryColor - Secondary elements
  • accentColor - Accent highlights
  • errorColor - Error messages
  • successColor - Success states

Gradients

  • loginBackground - Login screen background
  • cardBackground - Card elements background

Typography

  • titleFont - Page titles
  • headlineFont - Section headers
  • bodyFont - Body text
  • captionFont - Small text

How to build and archive app?

Development Build

  • Select your device or simulator
  • Press Cmd+R or click the Play button
  • Wait for build to complete
  • Test all authentication flows

Archive for Distribution

  • Select "Any iOS Device" as destination
  • Go to Product ‚Üí Archive
  • Wait for archive to complete
  • Organizer will open with your archive
  • Click "Distribute App"
  • Choose distribution method (App Store, Ad Hoc, etc.)

Before Distribution

  • Update version and build numbers
  • Test on multiple devices
  • Verify all authentication methods work
  • Check app icons and launch screen
  • Test in different languages

How to submit to App Store?

App Store Connect Setup

  • Go to App Store Connect
  • Click "My Apps" ‚Üí "+" ‚Üí "New App"
  • Fill in app information
  • Set up app categories and age rating
  • Upload app screenshots and description

Upload Build

  • Archive your app in Xcode
  • Click "Distribute App" in Organizer
  • Choose "App Store Connect"
  • Follow the upload process
  • Wait for processing (can take up to 24 hours)

Submit for Review

  • Select your processed build in App Store Connect
  • Complete all required app information
  • Add app review information and notes
  • Submit for review
  • Review process typically takes 24-48 hours
