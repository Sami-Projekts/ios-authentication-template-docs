üõ†Ô∏è Complete Setup Guide

Prerequisites

  • Xcode 15.0 or later
  • iOS 15.0+ deployment target
  • Swift 5.9 or later
  • Active Apple Developer Account
  • Google Developer Account (for Google Sign-In)

1. Firebase Setup

  1. Create Firebase Project
    1. Go to https://console.firebase.google.com/ 2. Click "Create a project" 3. Follow the setup wizard 4. Enable Google Analytics (optional)
  2. Add iOS App to Firebase
    1. Click "Add app" ‚Üí iOS 2. Enter your Bundle ID (e.g., com.yourcompany.authtemplate) 3. Download GoogleService-Info.plist 4. Drag the file into your Xcode project root
  3. Enable Authentication Methods
    1. Go to Authentication ‚Üí Sign-in method 2. Enable Email/Password 3. Enable Google (if using Google Sign-In) 4. Configure authorized domains
  4. Configure Firestore Database
    1. Go to Firestore Database 2. Create database in production mode 3. Set up security rules (see below)

    Firestore Security Rules:

    rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Users can only access their own data match /users/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; } // Prevent unauthorized access match /{document=**} { allow read, write: if false; } } }

2. Google Sign-In Setup

  1. Configure URL Scheme
    1. Open your project in Xcode 2. Select your target ‚Üí Info ‚Üí URL Types 3. Add URL Scheme from GoogleService-Info.plist 4. The scheme is the REVERSED_CLIENT_ID value
  2. Initialize in App
    // In your AuthenticationTemplateApp.swift import GoogleSignIn init() { FirebaseApp.configure() guard let path = Bundle.main.path(forResource: "GoogleService-Info", ofType: "plist"), let plist = NSDictionary(contentsOfFile: path), let clientId = plist["CLIENT_ID"] as? String else { fatalError("GoogleService-Info.plist not found") } GIDSignIn.sharedInstance.configuration = GIDConfiguration(clientID: clientId) }

3. Apple Sign-In Setup

  1. Enable Apple Sign-In Capability
    1. Select your target in Xcode 2. Go to Signing & Capabilities 3. Click "+ Capability" 4. Add "Sign in with Apple"
  2. Configure in Developer Portal
    1. Go to Apple Developer Portal 2. Certificates, Identifiers & Profiles 3. Select your App ID 4. Enable "Sign in with Apple" 5. Configure domain and email settings

üîß Troubleshooting

Common Issues & Solutions

Google Sign-In Not Working

  • Check REVERSED_CLIENT_ID in URL schemes
  • Verify GoogleService-Info.plist is added to project
  • Ensure Google Sign-In is enabled in Firebase Console
  • Test on physical device (not simulator)

Apple Sign-In Failing

  • Check Apple Sign-In capability is added
  • Verify App ID is configured in Developer Portal
  • Test on physical device (not simulator)
  • Ensure proper provisioning profile

Email Verification Not Working

  • Check spam/junk folder
  • Verify Firebase email templates are configured
  • Check authorized domains in Firebase Console
  • Ensure proper Firebase project setup

Build Errors

  • Clean build folder (Product ‚Üí Clean Build Folder)
  • Check Swift Package Manager dependencies
  • Verify minimum deployment target (iOS 15.0+)
  • Update to latest Xcode version

MVVM Architecture

The app follows the Model-View-ViewModel (MVVM) pattern for clean separation of concerns:

  • Models: Data structures (AppUser, AuthError)
  • Views: SwiftUI views for UI presentation
  • ViewModels: Business logic and state management
  • Services: External service integrations (Firebase, Google, Apple)
