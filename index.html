🎯 Getting Started

🎉 Two Ways to Get Started

Choose the approach that works best for you:

📱 Use Complete Template
🔧 Integrate into Existing Project

Option 1: Complete Template (Recommended)

Perfect for new projects or when you want a complete authentication system.

  1. Download the template and open in Xcode 15+
  2. Configure Firebase (see detailed setup below)
  3. Customize branding in AppConfig.swift and AppTheme.swift
  4. Run and test all authentication flows
  5. Build your app features on top of the authentication system

Option 2: Integrate into Existing Project

Add professional authentication to your existing iOS app.

  1. Copy core files (ViewModels, Models, Services, Utilities)
  2. Install dependencies (Firebase, GoogleSignIn)
  3. Integrate views into your app's navigation flow
  4. Configure Firebase and authentication providers
  5. Test integration and customize as needed

🛠️ Complete Setup Guide

Prerequisites

  • Xcode 15.0 or later
  • iOS 15.0+ deployment target
  • Swift 5.9 or later
  • Active Apple Developer Account
  • Google Developer Account (for Google Sign-In)

Step 1: Download and Open Project

📱 What you'll do:

Download the iOS Authentication Template from CodeCanyon and open it in Xcode. This template includes all the necessary files and structure for a complete authentication system.

🔧 Detailed Instructions:

  • Download the template ZIP file from your CodeCanyon purchase
  • Extract the ZIP file to your desired location
  • Open Xcode 15 or later
  • Navigate to the extracted folder
  • Double-click the .xcodeproj file to open the project

✅ What you'll see:

The project will open with a complete folder structure including Views, ViewModels, Models, Services, and Utilities folders, all organized following MVVM architecture.

Step 2: Access Package Dependencies

📦 What you'll do:

Navigate to Package Dependencies in Xcode to add the required Firebase and Google Sign-In packages.

🔧 Detailed Instructions:

  • In Xcode, select your project in the navigator
  • Click on your target under "TARGETS"
  • Go to the "Package Dependencies" tab
  • Click the "+" button to add a new package
  • You'll see the "Add Package Dependency" window

📋 Required packages:

You'll need to add these two packages in the next steps:

  • Firebase iOS SDK
  • Google Sign-In iOS

Accessing Package Dependencies in Xcode with + button

Step 3: Add Firebase iOS SDK

🔥 What you'll do:

Add the Firebase iOS SDK URL to your project to enable Firebase authentication and Firestore database functionality.

🔧 Detailed Instructions:

  • In the "Add Package Dependency" window, paste the Firebase URL
  • Enter: https://github.com/firebase/firebase-ios-sdk
  • Click "Add Package" and wait for Xcode to resolve the package
  • The package will be validated and ready for product selection
📋 Copy Firebase URL: https://github.com/firebase/firebase-ios-sdk

Entering Firebase iOS SDK repository URL

Step 4: Select Firebase Products

🎯 What you'll do:

Choose the specific Firebase products your authentication system needs.

🔧 Detailed Instructions:

  • After Xcode resolves the Firebase package, you'll see a list of available products
  • Select FirebaseAuth - Essential for authentication
  • Select FirebaseFirestore - For storing user data and usernames
  • Select FirebaseStorage - For profile pictures (optional)
  • Click "Add Package" to add these to your project

⚠️ Important:

Only select these three products to keep your app size manageable!

Selecting Firebase products: FirebaseAuth, FirebaseFirestore, and FirebaseStorage

Step 5: Add Google Sign-In SDK

🌐 What you'll do:

Add the Google Sign-In iOS SDK to enable Google authentication in your app.

🔧 Detailed Instructions:

  • Click the "+" button again to add another package
  • Enter the Google Sign-In iOS SDK URL
  • Paste: https://github.com/google/GoogleSignIn-iOS
  • Click "Add Package" and wait for resolution
  • Select the GoogleSignIn product in the next step
📋 Copy Google Sign-In URL: https://github.com/google/GoogleSignIn-iOS

✅ Verification:

After installation, you should see both Firebase and GoogleSignIn in your Package Dependencies list.

Adding Google Sign-In iOS SDK repository URL

Step 6: Create Firebase Project & Register iOS App

🎯 What you'll do:

Create a Firebase project and register your iOS app with the correct Bundle ID.

🔧 Detailed Instructions:

  • Go to https://console.firebase.google.com/ and create a new project
  • Click the iOS icon to add an iOS app
  • Enter your Bundle ID exactly as it appears in Xcode
  • Make sure the Bundle ID matches perfectly (case-sensitive)
  • Click 'Register app' to continue
📋 Copy Firebase Console URL: https://console.firebase.google.com/

📝 Critical Note:

The Bundle ID must be identical to your Xcode project's Bundle Identifier!

Selecting Google Sign-In products and completing package installation

Registering iOS app in Firebase with Bundle ID configuration

Step 7: Find Bundle ID in Xcode

🎯 What you'll do:

Locate your app's Bundle Identifier in Xcode to ensure it matches Firebase configuration.

🔧 Detailed Instructions:

  • In Xcode, select your project in the navigator
  • Click on your target under "TARGETS"
  • Go to the "General" tab
  • Find "Bundle Identifier" in the Identity section
  • Copy this exact value to use in Firebase

📝 Important Note:

This Bundle ID must match exactly between Xcode and Firebase - including capitalization and punctuation.

Finding Bundle Identifier in Xcode General settings

Step 8: Authentication Methods Overview

🎯 Was du tun wirst:

Sehen, welche Authentifizierungsmethoden wir in der Firebase Console aktivieren werden.

🔧 Detaillierte Anweisungen:

  • Navigiere zu Firebase Console → Authentication → Sign-in method
  • Du siehst eine Liste der verfügbaren Authentifizierungsanbieter
  • Wir werden drei Hauptmethoden aktivieren: Email/Password, Google und Apple
  • Diese drei Methoden bieten umfassende Authentifizierungsoptionen
  • Jede Methode wird in den folgenden Schritten konfiguriert

📝 Wichtiger Hinweis:

Email/Password bietet grundlegende Authentifizierung, Google ermöglicht einfache soziale Anmeldung und Apple Sign-In gewährleistet Datenschutz-Compliance.

Firebase Sign-in Methoden zeigen Email/Password, Google und Apple Authentifizierungsoptionen

Step 9: Enable Email/Password Authentication

🎯 Was du tun wirst:

Email- und Passwort-Anmeldung für deine Benutzer aktivieren.

🔧 Detaillierte Anweisungen:

  • Im Sign-in method Tab, finde den 'Email/Password' Anbieter
  • Klicke auf 'Email/Password' um es zu konfigurieren
  • Schalte den 'Enable' Switch ein
  • Optional aktiviere 'Email link' für passwortlose Anmeldung
  • Klicke 'Save' um die Änderungen zu übernehmen

📝 Wichtiger Hinweis:

Email/Password ist die grundlegendste Authentifizierungsmethode und funktioniert auf allen Plattformen.

Verfügbare Authentifizierungsmethoden in der Firebase Console

Step 10: Enable Google Sign-In

🎯 Was du tun wirst:

Google Sign-In Anbieter für einfache Benutzerauthentifizierung konfigurieren.

🔧 Detaillierte Anweisungen:

  • Finde 'Google' in der Sign-in Anbieter Liste
  • Klicke auf 'Google' um es zu konfigurieren
  • Schalte den 'Enable' Switch ein
  • Setze deine Projekt-Support-Email-Adresse
  • Klicke 'Save' um Google Sign-In zu aktivieren

📝 Wichtiger Hinweis:

Google Sign-In erfordert zusätzliche Einrichtung in Xcode mit URL-Schemas.

Aktivierung der Email/Password Authentifizierung in Firebase

Step 11: Enable Apple Sign-In

🎯 Was du tun wirst:

Apple als Sign-in Anbieter in der Firebase Console konfigurieren.

  • Finde 'Apple' in der Anbieter Liste und klicke darauf
  • Schalte den 'Enable' Switch ein um Apple Sign-In zu aktivieren
  • Konfiguriere deine App ID und Team ID vom Apple Developer Portal
  • Klicke 'Save' um Apple Sign-In Authentifizierung zu aktivieren

📝 Wichtiger Hinweis:

Apple Sign-In bietet das höchste Niveau an Benutzerdatenschutz und Sicherheit.

Aktivierung und Konfiguration des Google Sign-In Anbieters

Step 12: Verify Authentication Methods

🎯 Was du tun wirst:

Bestätigen, dass alle drei Authentifizierungsmethoden ordnungsgemäß aktiviert sind.

🔧 Detaillierte Anweisungen:

  • Überprüfe, dass Email/Password den Status 'Enabled' zeigt
  • Verifiziere, dass Google Sign-In den Status 'Enabled' zeigt

Aktivierung des Apple Sign-In Anbieters in Firebase

Step 19: Add GoogleService-Info.plist to Xcode

🎯 Was du tun wirst:

Die Firebase Konfigurationsdatei zu deinem Xcode Projekt hinzufügen und die REVERSED_CLIENT_ID kopieren.

🔧 Detaillierte Anweisungen:

  • Behalte diesen Wert - du brauchst ihn für die URL-Schema-Konfiguration

📝 Wichtiger Hinweis:

Die REVERSED_CLIENT_ID ist essentiell damit Google Sign-In ordnungsgemäß funktioniert.

Download der GoogleService-Info.plist Konfigurationsdatei

Step 20: Configure URL Scheme for Google Sign-In

🎯 Was du tun wirst:

URL-Schema in Xcode einrichten um Google Sign-In Callbacks zu handhaben.

🔧 Detaillierte Anweisungen:

  • In Xcode, wähle dein Projekt, dann dein Target

  • Klicke '+' um einen neuen URL Type hinzuzufügen
  • In 'URL Schemes', füge deine REVERSED_CLIENT_ID ein
  • In 'Identifier', gib 'google signin' als Beschreibung ein

📝 Wichtiger Hinweis:

Dieses URL-Schema ermöglicht es Google Sign-In nach der Authentifizierung zu deiner App zurückzukehren.

Hinzufügung von GoogleService-Info.plist zu Xcode und Kopieren der REVERSED_CLIENT_ID

🔧 Troubleshooting Guide

Common Issues & Solutions

🚫 Google Sign-In Not Working

Symptoms: Google Sign-In button doesn't respond or shows error

Solutions:

  • ✅ Verify REVERSED_CLIENT_ID in URL schemes matches GoogleService-Info.plist
  • ✅ Ensure GoogleService-Info.plist is added to project target
  • ✅ Check Google Sign-In is enabled in Firebase Console
  • ✅ Test on physical device (not simulator)
  • ✅ Verify Bundle ID matches exactly between Xcode and Firebase

🍎 Apple Sign-In Failing

Symptoms: Apple Sign-In shows error or doesn't appear

Solutions:

  • ✅ Check "Sign in with Apple" capability is added in Xcode
  • ✅ Verify App ID is configured in Apple Developer Portal
  • ✅ Test on physical device with valid Apple ID
  • ✅ Ensure proper provisioning profile is selected
  • ✅ Check Apple Sign-In is enabled in Firebase Console

📧 Email Verification Issues

Symptoms: Verification emails not received or links don't work

Solutions:

  • ✅ Check spam/junk folder in email client
  • ✅ Verify Firebase email templates are properly configured
  • ✅ Check authorized domains in Firebase Console
  • ✅ Ensure Firebase project is properly set up
  • ✅ Test with different email providers (Gmail, iCloud, etc.)

🔥 Firebase Connection Issues

Symptoms: App crashes on startup or Firebase functions don't work

Solutions:

  • ✅ Verify GoogleService-Info.plist is properly added to project
  • ✅ Check Bundle ID matches exactly in all configurations
  • ✅ Ensure Firebase.configure() is called in AppDelegate
  • ✅ Verify Firebase products are properly imported
  • ✅ Check Xcode console for specific Firebase error messages
